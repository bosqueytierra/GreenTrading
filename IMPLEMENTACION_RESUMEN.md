### 1. Crear la tabla en Supabase

Ejecutar el SQL de `DATABASE_MIGRATION_H1_M15.md`:

```sql
create table if not exists public.smc_h1_m15_setups (
  id bigint generated by default as identity primary key,
  symbol text,
  tipo_indice text,
  direccion text,
  fecha_detectada timestamptz default now(),
  zona_desde double precision,
  zona_hasta double precision,
  zona_size_puntos double precision,
  precio_actual_detectado double precision,
  precio_entrada_referencia double precision,
  score integer,
  evento text,
  ob boolean,
  fvg boolean,
  barrida boolean,
  estado text,
  tp_price double precision,
  sl_price double precision,
  ratio_rr double precision,
  max_reaccion_puntos double precision,
  resultado_puntos double precision,
  fecha_cierre timestamptz,
  motivo_cierre text,
  tendencia_h1 text,
  tendencia_m15 text,
  estrategia text default 'SMC PRO TENDENCIA H1 + CHOCH/BOS (M15)',
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

Crear índices:

```sql
create index if not exists idx_smc_h1_m15_symbol_estado 
  on public.smc_h1_m15_setups (symbol, estado);

create index if not exists idx_smc_h1_m15_created_at 
  on public.smc_h1_m15_setups (created_at desc);

create index if not exists idx_smc_h1_m15_tipo_indice 
  on public.smc_h1_m15_setups (tipo_indice);
```

## ⚠️ Importante

1. **Crear la tabla antes de usar la UI:** Sin la tabla `smc_h1_m15_setups`, la segunda pestaña no funcionará.

2. **Estados DESCARTADA para auditoría:** Las zonas que no cumplan validación se guardarán como DESCARTADA con `motivo_cierre` explicativo. Esto permite revisar por qué fueron rechazadas.

3. **Auto-refresh:** Cuando el auto-refresh está activo, usa la estrategia seleccionada en ese momento. Cambiar de pestaña reinicia el refresh para esa estrategia.

4. **Historial independiente:** Cada estrategia tiene su propio historial completo con métricas y contadores TP/SL por índice.
