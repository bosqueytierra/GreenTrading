# Migración Base de Datos: SMC PRO TENDENCIA H1 + CHOCH/BOS (M15)

## Nueva Tabla: `smc_h1_m15_setups`

Esta tabla almacena los setups de la nueva estrategia **SMC PRO TENDENCIA H1 + CHOCH/BOS (M15)**.

**IMPORTANTE:** No modificar ni reutilizar la tabla `smc_m15_setups` (usada por SMC M15 PRO).

---

## SQL para crear la tabla

```sql
create table if not exists public.smc_h1_m15_setups (
  id bigint generated by default as identity primary key,
  symbol text,
  tipo_indice text,
  direccion text,
  fecha_detectada timestamptz default now(),
  zona_desde double precision,
  zona_hasta double precision,
  zona_size_puntos double precision,
  precio_actual_detectado double precision,
  precio_entrada_referencia double precision,
  score integer,
  evento text,
  ob boolean,
  fvg boolean,
  barrida boolean,
  estado text,
  tp_price double precision,
  sl_price double precision,
  ratio_rr double precision,
  max_reaccion_puntos double precision,
  resultado_puntos double precision,
  fecha_cierre timestamptz,
  motivo_cierre text,
  tendencia_h1 text,
  tendencia_m15 text,
  estrategia text default 'SMC PRO TENDENCIA H1 + CHOCH/BOS (M15)',
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

---

## Columnas adicionales vs `smc_m15_setups`

La nueva tabla incluye las mismas columnas que `smc_m15_setups`, más:

- **estrategia**: Identifica la estrategia ("SMC PRO TENDENCIA H1 + CHOCH/BOS (M15)")

Esta columna permite distinguir entre estrategias si en el futuro se quisiera consolidar tablas.

---

## Estados posibles

Los estados son idénticos a los de SMC M15 PRO:

- **ACTIVA**: Zona detectada, esperando que el precio entre
- **EN_ZONA**: Precio dentro de la zona
- **PROFIT**: Precio salió de la zona favorablemente pero no alcanzó TP 1:1
- **TP**: Take Profit 1:1 alcanzado
- **SL**: Stop Loss alcanzado
- **PAUSADA**: Zona temporalmente pausada (otra zona activa para el mismo símbolo)
- **DESCARTADA**: Zona rechazada por no cumplir validación H1 + M15

## Trigger para updated_at

Mantener sincronizado el campo `updated_at`:

```sql
-- Función trigger
create or replace function public.handle_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

-- Trigger en la tabla
create trigger set_updated_at
  before update on public.smc_h1_m15_setups
  for each row
  execute function public.handle_updated_at();
```

## Notas importantes

1. **No tocar `smc_m15_setups`**: La tabla original debe quedar intacta
2. **Separación completa**: Cada estrategia usa su propia tabla
3. **Mismo esquema**: Facilita migración futura si se desea consolidar
4. **Column estrategia**: Permite identificación clara de la estrategia
