# SMC PRO TENDENCIA H1 + CHOCH/BOS (M15) - Procesador de Supabase

## Descripción

**smc_h1_m15_processor.py** es el procesador automático que:

1. ✅ Lee velas desde **public.market_candles** (misma fuente que SMC M15 PRO)
2. ✅ Ejecuta análisis SMC base con validación H1 + M15
3. ✅ Guarda resultados en **public.smc_h1_m15_setups**

**NO** crea un nuevo collector MT5.  
**NO** duplica velas.  
**Es** otro consumidor de las mismas velas ya existentes.

```sql

  id bigint generated by default as identity primary key,

  tipo_indice text,
  direccion text,
  fecha_detectada timestamptz default now(),
  zona_desde double precision,
  zona_hasta double precision,
  zona_size_puntos double precision,
  precio_actual_detectado double precision,
  precio_entrada_referencia double precision,
  score integer,
  evento text,
  ob boolean,
  fvg boolean,
  barrida boolean,
  estado text,
  tp_price double precision,
  sl_price double precision,
  ratio_rr double precision,
  max_reaccion_puntos double precision,
  resultado_puntos double precision,
  fecha_cierre timestamptz,
  motivo_cierre text,
  tendencia_h1 text,
  tendencia_m15 text,
  estrategia text default 'SMC PRO TENDENCIA H1 + CHOCH/BOS (M15)',
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

### 2. Velas en Supabase

El procesador requiere que **mt5_to_supabase.py** esté corriendo y haya recolectado velas:

- ✅ H1: mínimo 500 velas
- ✅ M15: mínimo 800 velas
- ⚠️ M1: opcional (solo si usas refinamiento M1)

Verificar velas en Supabase:


## Lógica del Procesador

### Flujo de procesamiento

Para cada símbolo, cada N segundos:

1. **Obtener velas** desde `public.market_candles`:
   - H1: últimas 500 velas
   - M15: últimas 800 velas
   - M1: últimas 200 velas (opcional)

2. **Analizar SMC** con `analyze_smc_h1_m15()`:
   - Detecta swings H1 y M15
   - Detecta BOS/CHOCH H1 y M15
   - Detecta FVG, OB, barridas en M15
   - Construye zona depurada M15
   - **VALIDA H1 + M15**:
     - Boom: H1 ALCISTA + Evento M15 ALCISTA
     - Crash: H1 BAJISTA + Evento M15 BAJISTA

3. **Gestionar zonas**:
   - Si zona válida y nueva:
     - Pausa zonas activas anteriores del mismo símbolo
     - Guarda zona como ACTIVA
   - Si zona inválida:
     - Guarda zona como DESCARTADA
   - Actualiza estados de zonas existentes (EN_ZONA, TP, SL, PROFIT)

### Estados de zona

| Estado | Descripción |
|--------|-------------|
| **ACTIVA** | Zona detectada, esperando entrada |
| **EN_ZONA** | Precio dentro de la zona |
| **PROFIT** | Precio salió de zona favorablemente (pero no TP 1:1) |
| **TP** | Take Profit 1:1 alcanzado |
| **SL** | Stop Loss alcanzado |
| **PAUSADA** | Zona pausada por nueva zona activa en mismo símbolo |
| **DESCARTADA** | Zona rechazada por validación H1+M15 |

### Lógica de pausado

**Una zona activa por símbolo:**

Cuando se detecta una nueva zona válida:
1. Se obtienen todas las zonas ACTIVAS del mismo símbolo
2. Se pausan todas las zonas activas anteriores
3. Se crea la nueva zona como ACTIVA

Ejemplo:

```
Boom 1000 Index:
- Zona 1: ACTIVA (detectada 10:00)
- [Nueva zona detectada 10:30]
- Zona 1: PAUSADA (pausada 10:30)
- Zona 2: ACTIVA (detectada 10:30)
```

## Troubleshooting

### ❌ "No hay velas para {symbol} {timeframe}"

**Causa:** `mt5_to_supabase.py` no está corriendo o no ha recolectado suficientes velas.

**Solución:**
1. Verificar que `mt5_to_supabase.py` esté corriendo
2. Esperar a que recolecte al menos:
   - 500 velas H1 (≈21 días)
   - 800 velas M15 (≈8 días)
3. Verificar en Supabase:
   ```sql
   select symbol, timeframe, count(*) 
   from public.market_candles 
   group by symbol, timeframe;
   ```

### ❌ "Error conectando a Supabase"

**Causa:** Credenciales incorrectas o red caída.

**Solución:**
1. Verificar `.env`:
   ```env
   SUPABASE_URL=https://...
   SUPABASE_ANON_KEY=eyJ...
   ```
2. Probar conexión:
   ```bash
   python test_supabase.py
   ```

### ⚠️ "Todas las zonas salen DESCARTADA"

**Causa:** Validación H1+M15 no se cumple.

**Razones comunes:**
- Boom con H1 BAJISTA → Descartado
- Boom con evento M15 BAJISTA → Descartado
- Crash con H1 ALCISTA → Descartado
- Crash con evento M15 ALCISTA → Descartado

**Solución:** 
1. Revisar logs para ver `razon_validacion`
2. Verificar que la lógica de validación sea correcta para tu caso
3. Si es correcto, esperar a que las condiciones se alineen

### 🔄 "Zona similar ya existe"

**Causa:** Detección de duplicados (misma dirección y rango similar).

**Solución:** Esto es normal y esperado. El procesador evita crear zonas duplicadas.

## Diferencias con smc_h1_m15_pro.py

| Aspecto | smc_h1_m15_pro.py | smc_h1_m15_processor.py |
|---------|-------------------|-------------------------|
| **Fuente de datos** | MT5 directo | Supabase (market_candles) |
| **Ejecución** | Manual (una vez) | Automática (loop continuo) |
| **Guardado** | No guarda | Guarda en smc_h1_m15_setups |
| **Estados** | Solo muestra análisis | Gestiona ACTIVA/EN_ZONA/TP/SL/etc |
| **Zonas pausadas** | No implementado | Sí (una activa por símbolo) |
| **Uso** | Testing/debug | Producción |

**Recomendación:**
- `smc_h1_m15_pro.py` → Para pruebas rápidas con MT5
- `smc_h1_m15_processor.py` → Para sistema en producción

## Mantenimiento

### Limpieza de zonas antiguas

```sql
-- Archivar zonas cerradas de hace más de 30 días
update public.smc_h1_m15_setups
set estado = 'ARCHIVADA'
where estado in ('TP', 'SL', 'PROFIT', 'PAUSADA', 'DESCARTADA')
  and fecha_cierre < now() - interval '30 days';

-- O eliminar directamente
delete from public.smc_h1_m15_setups
where estado in ('TP', 'SL', 'PROFIT', 'PAUSADA', 'DESCARTADA')
  and fecha_cierre < now() - interval '90 days';
```
